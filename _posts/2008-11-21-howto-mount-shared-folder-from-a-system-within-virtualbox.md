صراحة لأني من محبي ومستخدمي الـ VirtualBox ولكثرة إستعمالي له، أود أن أشرح بصورة بسيطة جداً كيف يمكنك أن تقوم بالوصول الى المجلدات الموجودة على نظامك الأصلي من داخل النظام الذي يعمل في البيئة التخيلية …

أول حاجة أذهب بعد تشغيل البرنامج VirtualBox وبعد ذلك من النظام الذي شغلته أذهب للأعلى في واجهة VirtualBox وأضغط على Devices، كما في الصورة التالية:

![](/assets/img/posts/2008/vbox-share2.png)

بعد ذلك أذهب وأضغط على Shared Folders ستظهر لك الشاشة التالية:

![](/assets/img/posts/2008/vbox-share1.png)

الآن نريد إضافة مجلد موجود على النظام الأصلي لكي نستطيع أن نعمل Access عليه من خلال النظام الذي يعمل بداخل VirtualBox. إذن قم بلأضغط على الأيقونة التي عليها إشارة + باللون الأخضر كما في الصورة السابقة، حيث ستظهر لك هذه الشاشة:

![](/assets/img/posts/2008/vbox-share3.png)

الآن قم بوضع المسار الذي تريد مشاركته مع النظام التخيلي، إذا لم تكن حافظ المسار أضغط على الأيقونة التي بجانب الحقل Folder Path ومن ثم قم بإختيار المسار المطلوب مشاركته … كما في الصورة التالية:

![](/assets/img/posts/2008/vbox-share4.png)

وبعد ذلك أعطي إسم للمجلد هذا لكي نستيطع التعامل معه بعد ذلك … مثلاً ضع في حقل Folder Name كلمة data … وأيضاً أترك الخيار Make Permanent كما هو … الآن أضغط على OK …

الآن كل ما عليك فعله هو عمل mount لهذا المجلد من داخل النظام الذي يعمل بداخل VirtualBox … طبعاً هناك طريقة لنظام جنو/لينوكس وطريقة لنظام الويندوز …

طريقة عمل mount للمجلد من داخل نظام جنو/لينوكس هي:

```
mount -t vboxsf folder_name mount_point
```

مثلاً نريد أن نعمل mount للمجلد الذي أضفناه data سنعمل التالي:

```
mount -t vboxsf data /mnt/data
```

الآن اذهب الى المجلد:

```
/mnt/data
```

ستجد الملفات التي قمت بمشاركتها مع النظام هذا ::20

طريقة عمل mount للمجلد من داخل نظام ويندوز هي:

```
start -> run -> cmd
```

بعد ذلك نفذ الأمر:

```
net use x: \\vboxsvr\folder_name
```

أي في مثالنا الذي بالأعلى سيتم ذلك من خلال:

```
net use x: \\vboxsvr\data
```
