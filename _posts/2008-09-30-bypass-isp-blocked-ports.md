السلام عليكم ورحمة الله وبركاته

كما وعدت أن أشرح كيفية تجاوز أي مزود خدمة، وأي بروكسي موجود عندك في بلدك، تابع معي هذا الموضوع البسيط لعمل ذلك.

أول شيء وأهم شيء هو أن يكون لديك حساب SSH خارجي، بدون هذا الحساب لن تنفع المسألة بتاتاً. إذا كنت لا تملك واحد بإمكانك شراء حساب من مواقع عدة، هذا أحدهم: [أضغط هنا](http://www.shellux.net/shx.services/shx.services.unix.shells.shtml)

الآن المسألة بسيطة للغاية، ما سنقوم به بالحقيقة هو إستعمال الإتصال بالسيرفر SSH لتمرير البيانات من خلالها، وبطبيعة الحال تشفيرها :) هذه الطريقة تسمى SSH Tunneling أو SSH Port Forwarding.

أول حاجة لنقوم بالإتصال بالسيرفر SSH من خلال الأمر:

```
ssh -D 8888 -C username@ip-address
```

أو

```
ssh -D 8888 -C username@domain.com
```

أستعمل أي منفذ أعلى من 1024 وذلك لانه المنافذ التي دون ذلك تحتاج الى صلاحيات root. ولهذا أستعملنا 8888 وبإمكانك وضع أي شيء تريد كما قلت لك أهم شي أعلى من 1024.

– الخيار D من أجل تخصيص منفذ داخلي للتمرير من خلاله.

– الخيار C لكي نستعمل الضغط مع الإتصال لأجل السرعة.

الآن لنتأكد من قيام جهازنا (127.0.0.1) من التنصت على المنفذ 8888، من خلال الأمر:

```
netstat -ntlp | grep 8888
```

أذا ظهرت لك مثل هذه:

```
tcp        0      0 127.0.0.1:8888          0.0.0.0:*               LISTEN      17889/ssh       
tcp6       0      0 ::1:8888                :::*                    LISTEN      17889/ssh
```

فهذا يعني إن الأمور تعمل للحين تمام التمام أكمل معي. الآن أذهب الى الفايرفوكس لديك وأذهب الى:

```
Edit --> Preferences --> Advanced --> Network --> Settings
```

وقم بإختيار:

```
Select Manual proxy configuration
```

وفي خانة SOCKS Host ضع: 127.0.0.1 والمنفذ 8888.

تأكد من إختيارك لـ SOCKS v5. وتأكد من وضع:

```
localhost, 127.0.0.1
```

للخيار No Proxy for.

الآن أفتح صفحة جديدة بالفايرفوكس وأذهب الى الرابط:

```
about:config
```

قم بالبحث عن الخيار:

```
network.proxy.socks_remote_dns
```

وأجعل قيمته true بدل من false. هذا الخيار ظروري لكي يكون الـ DNS Resolving من خلال الممر الذي عملنا وليس من خلال DNS الخاص بمزود الخدمة لديك …

الآن تصفحك للأنترنت سيكون مشفر وقادر على المرور وعبور مزودي خدمة الأنترنت

أيضاً من فوائد وجود حساب لك SSH على سيرفر خارجي هو إنك تستطيع تجاوز تخلف شركات مزودي خدمة الإنترنت التي تقوم بحجب المنفذ 25 وذلك بحجة ارسال SPAM منه. والحقيقة ومن خلال عملي السابق، عملت Site Survey على أكثر من مزود خدمة في المكان الذي أسكن فيه حالياً وكلهم يقومون بغلق المنفذ بسبب عدم إمتلاكهم الحلول وبسبب كون الـ IP Address’s Ranges المحددة لهم دائماً يتم عمل Black List لها وبالتالي ينتظرون فترة من الزمن لرفع الحضر هذا. هم عاجزون عن وضع حلول + هم مستخسرين في الناس وضع حلول، يعني ما ودهم يقومون بشراء حلول لهذه المشاكل وبس شاطرين بالفلسفة. على كل حال ألي يعرف وضعهم الحقيقي يحزن عليهم

أيضاً بإمكانكم عمل Remote Port Forwarding بالإضافة الى Local Port Forwarding. ربما إن سنحت الفرصة ولم أجد أحد يعرف كيف يعمل العبور لمنفذ 25 في الجدار الناري لمزود خدمة الأنترنت سأقوم بذكر الطريقة لكم.

أذن بالنهاية حاول تعمل:

– تشفير وتمرير للبريد الصادر الخاص بك وتجاوز المنفذ 25 لمزود خدمة الأنترنت.

– تشفير البريد الوارد الخاص بك.

– تشفير جميع إتصالك المستعمل بالـ Pidgin.

الله يرضى عليكم حاولوا تعملوها بأنفسكم.

**ملاحظة مهمة جداً:** أنا غير مسؤول عن أي إستعمال خاطيء لهذه الطريقة. الطريقة شرحتها لكي تحصل على حريتك الحقيقية بالتصفح المخفي + تجاوز الحجوبات المعمولة عليك لأسباب غبية، وليس لتصفح المواقع المحرمة أو استعمالها لأغراض خبيثة أو مؤذية لأي مسلم.**اللهم إني بلغت، اللهم فأشهد**.
